# Architecture Issue: Dual Terminal Execution Chaos

## Problem

Two competing terminal systems exist with overlapping responsibilities:

### System 1: Node.js Direct (Windows-Only)
```typescript
// src/app/api/terminal/route.ts
const shell = 'powershell.exe'  // Hardcoded!
const child = spawn(shell, ['-Command', command], {...})
```

### System 2: Python Backend (Cross-Platform)
```typescript
// src/app/api/python/terminal/route.ts
const response = await fetch(`${PYTHON_API_URL}/terminal/execute`, ...)
// Python backend uses OSDetector for Windows/Linux/Mac
```

### Consumer: Mixed Usage
```typescript
// src/tools/terminal/store.ts
try {
  // Try Python API first
  const pythonResponse = await fetch('/api/python/system/info')
} catch {
  // Fallback to... what exactly?
  // Code we just removed was trying direct terminal detection
}
```

## The Confusion

| Aspect | Node Terminal | Python Terminal |
|--------|--------------|-----------------|
| OS Support | Windows only | All platforms |
| Shell | Hardcoded PowerShell | Detected (PS/Bash) |
| Security | Basic | Command categorization |
| History | In-memory | Persistent |
| Used By | ??? | useTerminal() hook |

## Why This Is Bad

1. **Inconsistent behavior** - Different execution paths yield different results
2. **Maintenance burden** - Fix bugs in two places
3. **Windows-only limitation** - Node route prevents Mac/Linux usage
4. **Confusing stack** - Can't tell which system is active

## Solution

**Eliminate Node.js terminal route**, use Python exclusively:

```typescript
// Remove: src/app/api/terminal/route.ts
// Keep: src/app/api/python/terminal/route.ts (proxy to Python)

// Or better - direct fetch to Python from frontend:
const result = await pythonBridge.terminal.execute({command, cwd})
```

The Python backend already has:
- `OSDetector` for platform detection
- `ShellRunner` for cross-platform execution
- `ShellConfig` for command syntax per OS

## Files to Consolidate

- DELETE: `src/app/api/terminal/route.ts`
- KEEP: `src/app/api/python/terminal/route.ts`
- UPDATE: `src/tools/terminal/store.ts` - remove fallback logic
- ENSURE: Python backend always available
