# Architecture Issue: Mixed Data Flow Directions

## Problem

State lives in 5 different places with no clear ownership:

```
┌─────────────────────────────────────────────────────────────┐
│                     State Sources                           │
├─────────────────────────────────────────────────────────────┤
│ 1. React useState        │ Chat messages, UI state            │
│ 2. useRef (sync access)  │ messagesRef (for setTimeout)        │
│ 3. localStorage          │ Chat history, terminal config       │
│ 4. Python Backend        │ Terminal history, system info       │
│ 5. File System           │ rogius.config.json                  │
└─────────────────────────────────────────────────────────────┘
```

## The Sync Ref Problem

```typescript
// page.tsx lines 82-85
const messagesRef = useRef(messages)
useEffect(() => {
  messagesRef.current = messages  // Keep ref in sync
}, [messages])

// Why? Because setTimeout captures stale closures:
setTimeout(() => generateResponse([...messagesRef.current, msg]), 100)
```

This pattern indicates **state management is broken**. Using refs to bypass React's closure system is a code smell.

## The Python Backend State

```typescript
// Terminal state in TWO places:
// 1. React: const [commands, setCommands] = useState([])
// 2. Python: shell_runner.py maintains command_history

// Which is source of truth? Both get out of sync.
```

## Config Loading Chaos

```typescript
// page.tsx lines 109-114
loadConfigFromFile().then(fileConfig => {  // From rogius.config.json
  if (fileConfig) setConfig(fileConfig)
})

// But api-client.ts has:
const DEFAULT_CONFIG = {...}  // Hardcoded fallback
// And getStoredConfig() reads from localStorage
```

Three config sources: file, localStorage, hardcoded defaults. No clear priority.

## Why This Is Bad

1. **Race conditions** - Multiple sources update same concept
2. **Stale data** - Refs, state, storage all potentially different
3. **Bugs** - "Why didn't my change save?" Wrong source updated
4. **Mental model** - Developer must track which source is active

## Solution

Implement clear state ownership:

```typescript
// Single source of truth pattern
const StateSources = {
  CHAT_MESSAGES: 'react-state',        // Ephemeral, current session
  CHAT_HISTORY: 'localStorage',        // Persisted across sessions  
  TERMINAL_COMMANDS: 'python-backend',   // Python is executor, owns history
  SYSTEM_INFO: 'python-backend',       // OS detection happens there
  CONFIG: 'file-system',                 // rogius.config.json
}

// No refs needed - use proper state management (Zustand/Redux)
```

## Files Affected

- `page.tsx` - Multiple state sources intertwined
- `chat-storage.ts` - localStorage wrapper
- `terminal/store.ts` - React state + Python backend mix
- `config-loader.ts` - File loading
- `api-client.ts` - localStorage + defaults
