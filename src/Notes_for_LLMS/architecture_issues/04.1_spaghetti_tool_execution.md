# Architecture Issue: Spaghetti Tool Execution

## Problem

Tool definitions and execution are scattered across 4+ files:

### 1. Tool Definitions (System Prompt)
```typescript
// src/lib/api-client.ts lines 52-127
const DEFAULT_CONFIG: APIConfig = {
  systemPrompt: `You are Rogius...
AVAILABLE TOOLS:
- execute_command: Single command execution
- start_multistep_task, execute_next_step...`
}
```

### 2. Tool Schema (TERMINAL_TOOLS)
```typescript
// src/lib/api-client.ts lines 156-268
export const TERMINAL_TOOLS = [
  {
    type: 'function',
    function: {
      name: 'execute_command',
      description: '...',
      parameters: {...}
    }
  },
  // ... 6 more tools
]
```

### 3. Tool Types (Duplicated)
```typescript
// src/lib/api-client.ts lines 270-278
export interface ToolCall {
  id?: string
  index?: number
  type: 'function'
  function: { name?: string; arguments: string }
}

// src/lib/python-bridge.ts lines 75-85
// SAME structure, different naming:
interface AIChatResponse {
  tool_calls?: Array<{
    id?: string;
    index: number;
    type: string;
    function: { name: string; arguments: string }
  }>
}
```

### 4. Tool Execution (In Page Component)
```typescript
// src/app/page.tsx lines 244-261
if (toolCall.function.name === 'open_terminal') {
  assistantContent += '\n\n Terminal ready'
} else if (toolCall.function.name === 'execute_command') {
  const result = await terminal.executeCommand(args.command)
  // ... format output
}
// If-else chain for each tool
```

### 5. Parallel Python Implementation
```python
# src/tui/ai_client.py lines 69-189
TERMINAL_TOOLS = [...]  # Same tools, Python syntax
# Completely separate execution in tui.py
```

## Adding a New Tool Requires

1. Add to system prompt in `api-client.ts`
2. Add to TERMINAL_TOOLS schema in `api-client.ts`
3. Add interface in `api-client.ts`
4. Add execution handler in `page.tsx` (2 places: handleSubmit + generateResponse)
5. Add to Python TERMINAL_TOOLS in `ai_client.py`
6. Add handler in `tui.py`
7. Add types in `python-bridge.ts`

**7 files for 1 tool!**

## Why This Is Bad

1. **Inconsistent implementations** - Web and TUI diverge
2. **Missing tools** - Easy to forget to add in one place
3. **No validation** - No central registry to check completeness
4. **Code duplication** - Same tool logic written twice

## Solution

Centralized tool registry with code generation:

```typescript
// src/tools/registry.ts - Single source of truth
export const TOOL_REGISTRY = {
  execute_command: {
    description: 'Execute a shell command',
    parameters: {...},
    handler: 'terminal.executeCommand',  // Reference, not implementation
    platforms: ['web', 'tui']
  },
  start_multistep_task: {...},
  // All 7+ tools defined ONCE
} as const

// Generate Python version:
// tools.json → tools.py (code gen at build time)
```

## Recommended Structure

```
src/
  tools/
    registry.ts           # All tool definitions
    handlers/
      terminal.ts         # execute_command, open_terminal
      multistep.ts        # start_multistep_task, etc.
    types.ts              # Shared ToolCall types
```

## Impact

**Current**: 7 files to modify per tool
**Target**: 2 files (add to registry, add handler)
